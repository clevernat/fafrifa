# Data Card — Mid-latitude Extremes Indices for G6-1.5K-SAI / G6-1.5K-HiLLA

**Deliverable 2** of the Reflective SAI Uncertainty Database internship, Track A
(*Mid-latitude extreme winter weather response to SAI*). This card documents the
indices dataset produced by the `sai_extremes` workflow (Deliverable 1).

---

## 1. Summary

A multi-model set of annual ETCCDI-style climate extremes indices computed from
daily `tasmax`, `tasmin` and `pr` for two GeoMIP scenarios across up to four
Earth system models. The dataset is the quantitative basis for the revised
*Extratropical Circulation* uncertainty entry (Deliverable 3): it lets anyone
re-derive, on the Reflective Cloud Hub, how mid-latitude temperature and
precipitation extremes — and their inter-model spread — differ between
tropical-injection SAI and high-latitude (HiLLA) SAI.

| | |
|---|---|
| **Scenarios** | `G6-1.5K-SAI` (subtropical 30°N/30°S injection), `G6-1.5K-HiLLA` (high-latitude 60°N/S, low-altitude, seasonal injection) |
| **Models** | CESM2-WACCM6, MIROC-ES2H, UKESM1-0-LL, E3SMv3 *(HiLLA ensemble: UKESM, CESM, E3SMv3 — Duffey et al., 2026)* |
| **Variables in** | `tasmax`, `tasmin` (daily, °C), `pr` (daily, mm day⁻¹) |
| **Indices out** | 24 ETCCDI indices (16 fixed-threshold/block + 8 percentile-based) |
| **Temporal** | One value per calendar year over the analysis period |
| **Spatial** | Native model grid, regridded only if required; analysis focuses on NH mid-latitudes (30–70°N) |
| **Format (production)** | CF-1.10 NetCDF, one file per model × scenario |
| **Format (demo here)** | `.npz` per model × scenario + tidy `indices_regional_means.csv` |

> **Note on the demo files in `demo_output/`.** The Cloud Hub data could not be
> reached from the build environment, so the files here were generated by the
> *portable NumPy path* of the workflow on **synthetic** GeoMIP-shaped data
> (`scripts/generate_demo_dataset.py`). They reproduce the dataset's exact
> structure, variables, units and metadata, and the index values use the same,
> unit-tested kernels as production. They are **not** real model output and must
> not be used for science. To produce the real dataset, point
> `config/catalog.yaml` at the Cloud Hub and run
> `sai_extremes.pipeline.run_all(...)`, which writes CF-NetCDF.

## 2. Index list

Fixed-threshold & block indices (no base period required):

| ID | Long name | Var | Units |
|----|-----------|-----|-------|
| SU | Summer days (TX > 25 °C) | tasmax | days |
| ID | Icing days (TX < 0 °C) | tasmax | days |
| FD | Frost days (TN < 0 °C) | tasmin | days |
| TR | Tropical nights (TN > 20 °C) | tasmin | days |
| R10mm | Heavy precip days (≥ 10 mm) | pr | days |
| R20mm | Very heavy precip days (≥ 20 mm) | pr | days |
| CDD | Consecutive dry days (< 1 mm) | pr | days |
| CWD | Consecutive wet days (≥ 1 mm) | pr | days |
| TXx / TXn | Annual max / min of TX | tasmax | °C |
| TNx / TNn | Annual max / min of TN | tasmin | °C |
| Rx1day | Annual max 1-day precip | pr | mm |
| Rx5day | Annual max 5-day precip | pr | mm |
| PRCPTOT | Annual wet-day precip total | pr | mm |
| SDII | Simple daily intensity index | pr | mm day⁻¹ |

Percentile-based indices (require a base period; see §4):

| ID | Long name | Var | Units |
|----|-----------|-----|-------|
| TX90p / TX10p | Warm / cool days (TX > 90th / < 10th pct) | tasmax | % |
| TN90p / TN10p | Warm / cold nights (TN > 90th / < 10th pct) | tasmin | % |
| WSDI | Warm spell duration index (≥ 6 d > 90th pct) | tasmax | days |
| CSDI | Cold spell duration index (≥ 6 d < 10th pct) | tasmin | days |
| R95p / R99p | Precip from days > 95th / 99th wet-day pct | pr | mm |

The **winter-relevant** subset for Track A is the cold/temperature tail:
`FD, ID, TNn, TXn, TN10p, TX10p, CSDI`, plus precip `Rx1day, Rx5day, R95p, CWD`.

## 3. File schema

### 3.1 Production NetCDF (one file per model × scenario)

```
etccdi_<MODEL>_<SCENARIO>.nc   (CF-1.10)
  dimensions:  year, lat, lon
  coordinates: year(year) int; lat(lat) °N; lon(lon) °E in [-180,180)
  data vars:   SU, ID, FD, TR, R10mm, ... R99p   each (year, lat, lon)
    var attrs: long_name, units, description, cell_methods, index_id,
               base_percentile (percentile indices), bootstrap (temp pct indices)
  global attrs: title, model, scenario, Conventions, institution, source,
                references, baseline, history, sai_extremes_version
```

### 3.2 Demo `.npz` (portable preview)

```
etccdi_<MODEL>_<SCENARIO>.npz
  years   int   (nyear,)
  lat     float (nlat,)
  lon     float (nlon,)
  index__<ID>   float (nyear, nlat, nlon)   # one array per index
```

### 3.3 Tidy regional means (`indices_regional_means.csv`)

Columns: `model, scenario, index, region, year, value, units`. One row per
(model × scenario × index × region × year). Regions are cos(lat)-weighted means
over the boxes in `sai_extremes.regions` (`NH_midlat`, `Euro_Atlantic`,
`Mediterranean`, `Eurasia_high`, `Northern_Europe`). This is the analysis-ready
table for inter-model spread and SAI-vs-HiLLA contrasts.

## 4. Methods & provenance

* **Index definitions** follow ETCCDI (Zhang et al., 2011, *WIREs Clim. Change*;
  Klein Tank et al., 2009, WMO-TD/1500). Implemented as pure-NumPy kernels and
  wrapped for xarray; every kernel is unit-tested against hand-computed values
  (`deliverable-1-workflow/sai-extremes/tests/`).
* **Wet-day threshold**: 1 mm day⁻¹. **Percentile window**: centred 5-day
  calendar-day window. **Spell length**: ≥ 6 consecutive days (WSDI/CSDI).
* **Base period & bootstrap**: temperature percentile indices use a base-period
  climatology with the Zhang et al. (2005, *J. Climate*) out-of-base bootstrap to
  remove the in-base/out-of-base inhomogeneity. R95p/R99p use a single
  base-period wet-day percentile.
* **Choice of baseline (important).** Per Duffey & Irvine (2024), the surface
  response *attributed* to SAI depends on whether the reference baseline is a
  transient-warming state or a quasi-equilibrium state at the same global-mean
  temperature. The uncertainty metric is defined against *"a reference world with
  the same global-mean temperature but neither higher CO₂ nor SAI"*, so the base
  period should be a **quasi-equilibrium reference at the target GMT (~1.5 K)**,
  not a transient SSP segment. The chosen baseline is recorded in the catalog and
  in each file's global attributes.
* **Calendars** are preserved per model (`noleap`, `360_day`, `gregorian`); day-
  of-year handling in the percentile kernels is calendar-agnostic (1…366 with
  wrap-around). No silent calendar conversion is applied.
* **Units** are standardized on read (`K`→°C, `kg m⁻² s⁻¹`→`mm day⁻¹`).

## 5. Known limitations

* Demo values are synthetic (see §1 note) — structure only, not science.
* Indices are **annual** by ETCCDI convention; seasonal (DJF) circulation
  diagnostics are handled separately in the Weeks 7–8 analysis, not in this file.
* Inter-model comparison on native grids is valid for area-mean regional metrics;
  pointwise multi-model statistics require a common grid (regrid step, optional).
* Internal-variability sampling: single realisations give noisy percentile
  indices at gridpoint scale; regional/area-mean aggregation (the CSV) is the
  intended comparison level.
